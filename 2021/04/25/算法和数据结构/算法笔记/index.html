算法无关，记录一下

  小数转二进制

  小数部分乘2取整数部分，也就是0或1，顺序排序，得到的结果就是小数转二进制。4.125 转 二进制 就是，100.001。

  单精度浮点数二进制表达

  单精度浮点数32位，1位符号位，8位指数位，需要加127偏移量，23底数位。
  双精度浮点数64位，1位符号位，11位指数为，需要加1023偏移量，52底数位。

4.125 -> 100.001 
      -> 1.00001 * 10^2 -> 1.00001 * 10^10
其中 00001 就是底数，2就是指数，需要加上127偏移量。
2 + 127 -> 10000001
4.125 -> 0 10000001 00001000000000000000000

为什么要加上偏移量：计算机中使用补码表示数，如果指数的8位 还是补码表示，那么会使得 浮点数中有两个符号位，多了一个指数的符号位。因此加上偏移量的结果，也就是 移码 ，来实现无符号整数表示。这样在比较浮点数大小时，就只需要判断一个符号位即可。指数的8位，可以直接无符号的判断大小。
为什么偏移量的值是127：8位无符号的值范围是 0 ~ 255 (00000000 ~ 11111111)，其中 00000000 和 11111111 移码，在 指数中不不使用，因此指数的范围是 1 ~ 254 (00000001 ~ 11111110) 共 254个值。因此使用254的中位数，127 或 128作为偏移量，是合理的。
  如果以 128 为偏移量，那么 在指数值为 +127 (01111111)时，结果为 11111111，因为不使用该值，相当于溢出。因此以 128 为偏移量，指数的范围是，-127 ~ 126。
  如果以 127 为偏移量，那么 在指数值为 -127 (10000001)时，结果为00000000，因为不适用该值，相当于溢出。因此以 127 为偏移量，指数的范围是，-126 ~ 127。 

当23位底数全为1时，底数值最大
1.11111111111111111111111 = 1.9999998... 约为 2
当23位底数全为0时，底数值最小
1.00000000000000000000000 = 1  

因此128偏移量，浮点数的取值范围是：
-2 * 2^126 ~ -1 * 2^-127 和 1 * 2^-127 ~ 2 * 2^126。
数量为 ： 2 * ( 2 * 2^126 - 1 * 2^-127 ) 约为 2 ^ 128。

因此127偏移量，浮点数的取值范围是：
-2 * 2^127 ~ -1 * 2^-126 和 1 * 2^-126 ~ 2 * 2^127。  
数量为 ： 2 * ( 2 * 2^127 - 1 * 2^-126 ) 约为 2 ^ 129。

  由此可得，当偏移量为127时，取值范围更大。精度越大则取值范围越小，精度越小则取值范围越大。标准规定偏移量值为127。

整型边界值

  在int整型范围内，负数 转 正数可能会越界，正数 转 负数，是不会越界的。因此，当题目中条件涉及到正负数计算时，可以根据条件，把所有正数 转为 负数 计算，在统一都为负数的条件下，进行计算，只需要考虑 结果的正负，以及结果的越界情况。如果判断一个值 乘 A的绝对值，是否越界，可以先将边界值 除 A的绝对值。

数学运算

  字符串的乘法加法

int res = num1 + num2 + carry;或者 res = num1 * num2 + carry;
carry = res / 10;
res = res % 10;

  字符串运算中，不管乘法还是加法 ，上面这部分是通用的逻辑。
  在乘法中，长度为n的数 和 长度为m短的数 相乘，那么结果的长度不会超过 n + m，不会低于 m + n - 1。因此字符串乘法时，可以利用该特点，做辅助数组，保存每一字符相乘结果 到 数组中对应位置。如，n中 i上的字符，和m中j上的字符相乘，因为字符相乘结果最大81，不会超过2位数，那么对应的结果就是在，i+j和 i+j+1，这两个位置上。也就是 res/10 在 i+j位置，res%10在i+j+1位置。

  减法实现除法

  以被除数100，除数3为例。100 减 4 次 3 和 100 减 1 次 4*3，是一样的结果。因此可以通过左移除数，也就是 3 << 2 或者 直接 除数乘4，来减少循环次数。需要注意的是，减法的次数 同样需要随着除数的同步移动, 也就是 减 1 次的这个 1， 1 << 2。

  乘法实现x的n次幂

  和减法实现除法思路相似，不使用暴力法一个一个计算。直接计算n/2次幂，再将结果相乘。 

res = x^(n/2) 
x^n =  res * res 或者 x^n = res * res * x
注意奇偶

堆

  建堆的时间复杂度

  建堆时，当前节点向上移动，也就是说父节点向下移动。对于长度为N的数组，高度为H；H = log2(N) + 1：
原最后一层的节点 下移的次数 为0，原倒数第二层 最多下移一次，以此类推。
因此 总时间复杂度 为 :
假设当前高度为X，该层共有 2^(X - 1) 个节点
倒数第一层所有节点时间 + 倒数第二层所有节点时间 + …. + 第一层时间所有节点时间
2^(H - 1) * 0 + 2^(H - 2) * 1 + … + 2^(0) * (H - 1) = 2N - 2 - log2(N) = O(N);
因此 时间复杂度为 O(N)

分治

  分治处理 最小K个数 的时间复杂度计算

最小k个数(TopK),使用分治思路的时间复杂度;
  遍历完一遍后,O(N);如果 index != k -1;则继续分区,第二次遍历平均 O(N/2);
因此时间复杂度为 O(N) + O(N/2) + O(N/4) +…. = O(2N) = O(N);

中位数 — 寻找两个正序数组的中位数

  中位数的特点

  1.该位置左边的长度 等于 该位置右边的长度。2.左边的最大值 小于等于 右边的最小值。

  找第K个数

  实现方法1，在两个有序数组中，找第K个数，只是K在该场景下等于 长度/2，也就是中位数是特殊的第K个数。每次循环 比较两数组 第K/2 上的数，并排除K/2个。当K为1时，两数组中，小的那个就是第K个数。该方法时间复杂度 O(logK)。但是没有用到中位数的特性，只是把中位数当 第K个处理。

  数组区域划分

  实现方法2，在两个数组1和2中找中位数，将两个数组分别划分为左右两个区域。左区域个数根据len1和len2计算出。假设 i为数组1上划分左右的坐标，j为数组2上划分左右的坐标。得i + j = 左区域个数。然后再根据，中位数特性2，找到符合条件的位置，就能找到中位数。时间复杂度 O(log(min(len1,len2)))

回文串

  回文串的特点

  1.从左右两个向内遍历，两边字符相同。2.从中间向两边遍历，遍历字符相同。3.回文串去掉最左边及最右边，中间部分依旧是回文串。

  判断回文串的方式

  1.从左右向内遍历。2.从中间向外遍历。3.在左右两边相等的情况下，判断中间部分是否为回文串。因为回文串长度有奇偶之分，因此从i点向左右两边判断是否是回文串时，除了left,right = i以外，还需要判断left = i，right = i+1 判断偶数时的情况。

递归、回溯

  自己调用自己，也就是递归，是算法结构；多种选择，选择其中一种，根据条件退回，再选择，也就是回溯，是算法思想。回溯在题目中的体现，一般就是 添加一个值，相当于选择，去掉添加的值，相当于回退，在添加下一个值。算法题中涉及的回溯，最常见的就是上面的这个流程。递归 + 回溯 一般就是 DFS 深度优先遍历。 当题目中出现 “所有符合条件的结果” 这类情况时，一般就是 dfs + 根据条件剪枝。根据题目画出树，会更加清晰。

接雨水

  接雨水提，如果是计算i位置垂直容量，那么涉及 min(i左边最大高度，i右边最大高度)，因此需要双指针，或者2个辅助数组保存i位置的左右最大高度。如果计算i 和 j之间的水平容量，使用单调递减栈计算。

  如图所示，假设右边存在一个比左边都高的高度，可以使得左区域的位置i，垂直容量只需要考虑i位置时的leftMax。因此一个变量来保存并更新leftMax即可。同理，如果左边存在一个比右边都高的高度，右区域只需要考虑rightMax。

  如图所示，当存在 left高度 小于 right高度时，那么right位置，就相当于上面提到的，比左区域都大的高度。那么left位置能存储的雨水，只需要考虑， 0 ~ left位置的 leftMax。同理，如果存在 left高度 大于 right高度时，right位置能存储的雨水，只需考虑，right ~ length - 1位置的 rightMax。
